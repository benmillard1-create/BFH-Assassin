BFH Assassin — Sprint 2

WHAT THIS VERSION ADDS
- 4-digit PIN for every player
- Multiple players can share one phone or iPad
- Player login by game code, name and PIN
- Automatic random targets, objects and rooms
- Private mission page
- Public status board
- Public event feed
- Ghost status after elimination
- One mission only; targets never change

INSTALL IN THIS ORDER
1. In Supabase, open SQL Editor.
2. Create a new query.
3. Paste and run all of supabase_sprint2.sql.
4. In this project, open config.js.
5. Paste your Supabase anon key in place of PASTE_YOUR_ANON_KEY_HERE.
6. Replace the old GitHub files with:
   index.html
   styles.css
   app.js
   config.js
7. Commit the changes.
8. Refresh your GitHub Pages website.

IMPORTANT
- Use at least 3 players.
- Each player must remember their own 4-digit PIN.
- The public status board never reveals targets, rooms, objects or PINs.
- This is a family prototype. The security model is suitable for a private family game, not for sensitive data.
